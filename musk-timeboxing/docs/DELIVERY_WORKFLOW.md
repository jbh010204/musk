# Delivery Workflow

사용자 요청을 처리할 때 아래 고정 플로우로 전달합니다.

## 1) 구현

- 실제 코드 변경을 먼저 수행
- 기능별로 변경 파일을 묶어 설명

## 2) 리뷰

- 기능/UX/회귀 관점에서 결과 점검
- 발견 이슈가 없으면 `없음`으로 명시
- 잔여 리스크(있다면) 한 줄로 명시

## 3) 검증

- 최소 실행:
  - `npm run lint`
  - `npm run build`
- DnD/모달/저장/날짜 이동 영향 시:
  - `npm run test:e2e`

## 4) 다음 작업 추천

- 우선순위 3개 제안
- 각각 기대효과를 한 줄로 설명

## Response Template

```md
구현

리뷰
- ...

검증
- ...

다음 작업 추천
1. ...

3. ...
```
